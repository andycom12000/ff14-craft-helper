# CONTEXT — 吐司工坊 Domain Glossary

> 純術語表。不放實作細節、不當 spec、不當 scratch pad。

## 製作能力與資源耦合

- **作業（Craftsmanship）** — 推進「進度（Progress）」的能力。進度沒到 difficulty 就做不出成品。
- **加工（Control）** — 推進「品質（Quality）」的能力。品質決定能否 HQ。
- **耐久（Durability）+ CP** — 進度步驟與品質步驟**共用**的單一資源預算。

**關鍵關係：作業與加工不是兩條獨立的軸，而是同一份耐久＋CP 預算的分配。** 補作業讓進度用更少步數/CP/耐久完成，省下的資源可轉去衝品質——所以作業能**間接**提升品質。因此「該補作業還是加工」不是對錯問題，而是**哪種分配最省 gil** 的問題：直接補加工買品質通常遠比繞道補作業便宜。

## 鑲嵌建議（Meld Advice）

工具對「裸裝差一點就能 HQ」情境推薦的鑲嵌方案。其正確性合約為：

1. **保證 HQ** — 方案套用後 solver 能 double-max（進度滿 ＋ 品質滿）。
2. **最省 gil** — 在所有能保證 HQ 的方案中，總 gil 成本最低（涵蓋作業/加工耦合下的分配取捨）。
3. **符合現實槽位** — 鑲嵌總量不得超過裝備實際可鑲格數（`SLOT_STRUCTURE`：guaranteed 18 ＋ overmeld 42 ＝ 60）。

僅滿足 (1) 不足夠——一個鑲在昂貴繞路、數量爆量但「技術上能 HQ」的方案，不符合合約。

## 鑲嵌工作台（Meld Workbench）

鑲嵌建議的統一框架，兩種模式共用同一個正向 solver substrate（`solveCraftForRecipe` / `simulateCraftForRecipe`）：

- **正向（試算台）** — 使用者自擺鑲嵌（grade／顆數）→ 即時看素質與能否 HQ。對齊社群工具，但多了真 solver 自動找手法。
- **逆向（最佳化）** — 給目標 HQ，自動反推**最省鑲嵌**（見上方「鑲嵌建議」合約）。本質是在候選鑲嵌素質上反覆呼叫正向 substrate 挑最省——逆向是正向的搜尋外殼，非另一套引擎。

逆向結果可載入正向試算台微調，形成「給答案 → 驗證調整」閉環。

## 鑲嵌格與成功率

- **保證格（Guaranteed slot）** — 100% 成功的自然綠槽，免 fail-ladder 成本。
- **過量鑲嵌格（Overmeld slot）** — 成功率遞減（`OVERMELD_SUCCESS_LADDER`），每填 1 格的期望購買顆數隨深度上升（約 6~20 顆）。「期望購買顆數」與「占用格數」是兩個量。
